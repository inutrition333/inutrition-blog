# 愛營養部落格 — 操作手冊

> 這份手冊是給 Sunny 或團隊裡負責發文的人看的。
> 所有操作都在瀏覽器裡完成，不需要打開程式碼。

---

## 1. 怎麼進入後台？

打開瀏覽器，輸入：

```
https://blog.inutrition.com.tw/admin
```

第一次進入會要求用 GitHub 帳號登入。登入一次之後瀏覽器會記住，下次直接進後台。

---

## 2. 怎麼發一篇新文章？

1. 進後台 → 左邊選「**部落格文章**」
2. 右上角點「**新增 部落格文章**」
3. 填表單（每個欄位下面都有提示文字）：

| 欄位 | 說明 | 範例 |
|---|---|---|
| 文章標題 | 吸引人的標題 | 苦瓜胜肽跟山苦瓜粉差在哪裡？ |
| 分類 | 下拉選一個 | 代謝循環 |
| 發布日期 | 點日曆選 | 2026-04-14 |
| 閱讀時間 | 大概抓就好（分鐘） | 6 |
| 主圖 | 點「選擇圖片」上傳（選填） | — |
| 文章摘要 | 首頁列表用的短描述 | Sunny 帶你看懂三者差別⋯ |
| 活動識別碼 | 追蹤產品連結成效 | 苦瓜胜肽_202604 |
| 精選文章？ | 打勾 → 首頁置頂 | — |
| 草稿？ | 打勾 → 不會出現在網站 | — |
| 文章內文 | 用 Markdown 寫 | 見下方 |

4. 寫完後按右上角「**發布**」
5. 等大約 1-2 分鐘，Vercel 會自動重新部署，新文章就上線了

## 3. 怎麼在文章裡加「產品卡片」？

在文章內文裡，插入下面這段（複製貼上，改裡面的值就好）：

```
:::product
image: /images/uploads/你的產品圖.jpg
title: 產品名稱
description: 一兩句話描述這個產品跟文章的關聯
meepshopUrl: https://www.inutrition.com.tw/products/產品ID
utmContent: 產品名_文末
:::
```

**每個欄位說明：**

- `image:` — 產品圖路徑（先在後台上傳圖片，複製路徑貼過來）
- `title:` — 產品名稱，例如「穩穩棒棒堂 — 花蓮 4 號山苦瓜 + 苦瓜胜肽 PPK」
- `description:` — 簡單描述產品跟文章主題的關聯
- `meepshopUrl:` — MeepShop 產品頁的網址（不用加 utm，系統會自動加）
- `utmContent:` — GA 追蹤用，建議格式「產品名_位置」，例如「穩穩棒棒堂_文末」

⚠️ **注意**：`:::product` 和 `:::` 的前後都要空一行，不然會顯示不出來。

## 4. 怎麼在文章裡加「延伸閱讀」？

```
:::related{slug=另一篇文章的網址名稱}
另一篇文章的標題
:::
```

**範例：**

```
:::related{slug=bitter-melon-peptide-vs-powder}
苦瓜胜肽跟山苦瓜粉差在哪裡？弄懂這件事，挑選會更有方向
:::
```

- `slug=` 後面填的是目標文章的**網址名稱**（就是 `/posts/` 後面那串英文）
- 第二行填目標文章的標題（讓讀者知道連過去會看到什麼）

## 5. 怎麼在文章裡加「Sunny 小叮嚀」？

```
:::sunny
你想放在綠色框框裡的文字。
可以寫很多行。
**粗體**也可以用。
:::
```

## 6. 怎麼換分類？

後台表單裡有個「分類」下拉選單，直接選就好。目前有六個分類：

1. 體態管理
2. 腸胃保健
3. 代謝循環
4. 美容養顏
5. 睡眠放鬆
6. 銀髮族保養

⚠️ 要新增或刪除分類 → 需要請工程師同時改兩個檔案（程式碼裡的 config），沒辦法在後台加。

## 7. 怎麼找自己發過的文章修改？

1. 進後台 → 左邊「部落格文章」
2. 看到所有已發布的文章列表
3. 點進去就可以編輯
4. 改完按「**發布**」（即使是修改舊文，也是按發布）

## 8. 網址格式規則

每篇文章的網址長這樣：

```
blog.inutrition.com.tw/posts/bitter-melon-peptide-vs-powder
```

`/posts/` 後面那串叫 **slug**（網址名稱）。建議規則：

- 用**英文、數字、短橫線 `-`**
- 不要用中文（雖然技術上可以，但網址會變成 %E8%8B%A6... 很醜）
- 不要用空格或底線
- 越短越好，但要看得懂意思

好的範例：`bitter-melon-peptide-vs-powder`、`mulberry-leaf-extract-explained`
不好的範例：`article-1`、`苦瓜`、`sunny_note_about_bitter_melon_20260414`

## 9. 圖片怎麼放？多大適合？

**上傳方式：** 在後台編輯文章時，點「主圖」欄位的「選擇圖片」就能上傳。

**建議規格：**

| 項目 | 建議 |
|---|---|
| 寬度 | 1200px |
| 格式 | JPG 或 WebP（不要用 PNG，檔案太大） |
| 每張大小 | < 500KB |
| 產品圖 | 正方形或 4:3 比例最好看 |

圖片上傳後會存在 GitHub repo 的 `public/images/uploads/` 資料夾。

## 10. Markdown 常用語法速查

| 你想做的事 | 怎麼寫 | 顯示效果 |
|---|---|---|
| **粗體** | `**粗體文字**` | **粗體文字** |
| *斜體* | `*斜體文字*` | *斜體文字* |
| 大標題（H2） | `## 標題文字` | 文章目錄會自動抓 |
| 小標題（H3） | `### 小標題` | 暖棕色小標題 |
| 超連結 | `[顯示文字](https://網址)` | 點了會跳轉 |
| 項目清單 | `- 第一項` | • 第一項 |
| 編號清單 | `1. 第一項` | 1. 第一項 |
| 分隔線 | `---` | 水平線 |

---

## 11. 什麼時候要請工程師幫忙？

以下這些事**沒辦法在後台做**，需要改程式碼：

- 新增 / 刪除 / 改名分類
- 改網站版型（顏色、字體、排版）
- 改產品卡片的按鈕文字或樣式
- 加新功能（例如搜尋、留言、電子報訂閱）
- 改網域設定

日常發文、改文章、上傳圖片、換分類 → 全部在後台搞定，不用找工程師。
